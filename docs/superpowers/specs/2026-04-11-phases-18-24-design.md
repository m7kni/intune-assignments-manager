# Phases 18–24 Implementation Spec

## Overview
Implement all 7 remaining phases from todos.txt in parallel using isolated worktrees, then integrate shared-file modifications (navigation, wizard, execute) in a final pass.

## Execution Strategy

### Wave 1 (parallel worktrees)
- **Phase 18**: Windows Update Ring Management (4 update sub-types)
- **Phase 20**: Remediation Scripts (Proactive Remediations)
- **Phase 22**: Windows Autopilot Management
- **Phase 24**: Platform Scripts Management

### Wave 2 (parallel worktrees)
- **Phase 19**: Policy Comparison & Diff Tool
- **Phase 21**: Bulk Device Actions
- **Phase 23**: Enhanced Reporting & Export

### Wave 3 (sequential, main branch)
1. Merge all feature branches
2. Navigation restructure (grouped sidebar sections)
3. Wizard integration (new tabs in StepSelectItems, wizard.ts state, execute.ts handlers)
4. Full build + lint verification
5. Mark all phases complete in todos.txt

## Architecture Decisions
1. All phases follow the established types → schemas → API → utility → pages → components pattern
2. Shared files (layout, wizard, execute, merge) are NOT modified by feature agents — only during integration
3. Script viewer functionality (Phases 20 & 24) built independently per phase, consolidated during integration if warranted
4. Phase 22 (Autopilot) adds DeviceManagementEnrollment.Read.All to the permission system
5. Phase 19 (Policy Comparison) builds a new comparison utility in src/lib/utils/policy-compare.ts
6. Phase 23 (Reports) aggregates existing Graph calls — no new API endpoints needed
7. Navigation restructured into: Home, Management, Devices, Tools, Settings groups
